1. Overview

The CodeRed NDR is a Network Detection and Response (NDR) appliance with built-in IDS (Intrusion Detection System) and optional IPS (Intrusion Prevention System) capabilities. It passively monitors your network traffic for threats, anomalies, and suspicious behavior using Zeek (behavioral metadata and UEBA) and Suricata (signature-based IDS/IPS engine with 50,000+ rules). In IDS mode, the sensor detects and alerts on malicious traffic. In IPS mode (inline deployment), it can actively block threats in real time. All alerts, network metadata, and detection events are forwarded to your central CodeRed AI platform for analysis and response.

The sensor uses multiple network interfaces:

Multi-Zone Monitoring
You can add multiple monitoring interfaces to capture traffic from different network zones, VLANs, or switch segments simultaneously. The sensor automatically distributes Zeek workers and Suricata capture threads across all monitoring interfaces.

4. Installation

CodeRed NDR installs on any Ubuntu 22.04 or 24.04 server — on-premise, cloud, or virtual machine.

One-Line Install

# SSH into your Ubuntu server, then run:
curl -sSL https://raw.githubusercontent.com/lolomee/CodeRed-NDR/main/install.sh | sudo bash

The installer automatically:

  1. Installs Zeek (network metadata engine)
  2. Installs Suricata (IDS/IPS engine with 50,000+ rules)
  3. Installs Filebeat (log forwarder to CodeRed AI)
  4. Installs the CodeRed NDR management CLI
  5. Creates the coderedai management user
  6. Applies security hardening (SSH, firewall, AppArmor, fail2ban, kernel)

All NDR services are installed but not started — you start them after configuring the sensor.

After Installation

# SSH in as the management user:
ssh coderedai@<your-server-ip>
Password: coderedai

You'll see the management menu. Follow the setup steps in Section 7.

5. Network Mirror / SPAN Configuration

The sensor needs to receive a copy of your network traffic via a SPAN/mirror port on your network switch.

What to Mirror
At minimum, mirror your internet uplink (the port connecting to your firewall/router). This captures all north-south traffic. For east-west visibility, also mirror inter-VLAN trunk ports and server farm ports.

Cisco IOS

monitor session 1 source interface GigabitEthernet0/1 both
monitor session 1 destination interface GigabitEthernet0/24

Cisco Nexus

monitor session 1 type span
  source interface Ethernet1/1 both
  destination interface Ethernet1/48
  no shut

Arista EOS

monitor session 1 source Ethernet1 both
monitor session 1 destination Ethernet48

Juniper EX / QFX

set forwarding-options analyzer SPAN input ingress interface ge-0/0/0.0
set forwarding-options analyzer SPAN input egress interface ge-0/0/0.0
set forwarding-options analyzer SPAN output interface ge-0/0/47.0

HP / Aruba

mirror-port 48
interface 1 monitor

MikroTik

/interface ethernet switch
set switch1 mirror-source=ether1 mirror-target=ether24

VMware vSwitch / VDS

If monitoring virtual machines on the same hypervisor, configure port mirroring on the VDS, and enable Promiscuous Mode: Accept on the monitoring port group.

Using a Network TAP

For production environments, a hardware TAP (Garland, Gigamon, Dualcomm) is more reliable than SPAN — no packet drops under load.

6. Cloud Deployment (AWS / Azure / Alibaba Cloud)

In cloud environments, use the provider's traffic mirroring feature instead of a physical SPAN port.

AWS

  1. Launch an Ubuntu 22.04/24.04 EC2 instance with 2+ NICs (management + monitor)
  2. Run the CodeRed NDR install script
  3. Create a VPC Traffic Mirror Target pointing to the sensor's monitor ENI
  4. Create a Traffic Mirror Session from source instance(s) to the target
aws ec2 create-traffic-mirror-target --network-interface-id eni-SENSOR-MONITOR --description "CodeRed NDR"
aws ec2 create-traffic-mirror-session --network-interface-id eni-SOURCE --traffic-mirror-target-id tmt-xxx --traffic-mirror-filter-id tmf-xxx --session-number 1

Azure

  1. Launch an Ubuntu VM with 2+ NICs, enable Accelerated Networking
  2. Run the CodeRed NDR install script
  3. Configure Virtual Network TAP (vTAP) targeting the sensor's monitor NIC

Alibaba Cloud

  1. Launch an ECS instance with 2+ ENIs
  2. Run the CodeRed NDR install script
  3. Configure VPC Traffic Mirroring from source ENI(s) to sensor monitor ENI

7. Sensor Setup

After installation, SSH in as coderedai to configure the sensor:

ssh coderedai@<your-server-ip>
Password: coderedai

The management menu shows a setup checklist at the top. Complete these steps:

  1. Option 5 — Network settings: Set static IP, gateway, DNS (skip if already configured)
  2. Option 6 — Hostname: Set a descriptive name (e.g., hq-sensor-01)
  3. Option 7 — Monitor interfaces: Select which NIC(s) receive SPAN traffic
  4. Option 8 — CodeRed AI destination: Enter the IP and port of your CodeRed AI platform
  5. Option 12 — Test monitor interfaces: Verify packets are arriving on the monitor NIC(s)
  6. Option 13 — Test CodeRed AI connection: Verify the sensor can reach your CodeRed AI platform
  7. Option 9 — Start NDR: Start Zeek, Suricata, and Filebeat

Once all checklist items show [✓], the sensor is active and monitoring.

Change default password
Use option 15 to change the default coderedai password after setup.

9. Troubleshooting

Sensor Is Not Receiving Traffic

  1. Test monitor interfaces: Menu option 12 — runs a live packet capture test
  2. Check SPAN on your switch: Verify the session is active
  3. VMware users: Verify Promiscuous Mode: Accept on the port group
  4. Check interface status: Menu option 2 — verify monitor NICs show UP

Sensor Cannot Reach CodeRed AI

  1. Test connection: Menu option 13 — tests DNS, TCP port, and service response
  2. Check firewall rules: Ensure port 9200 (or your configured port) is open from sensor to CodeRed AI
  3. Verify CodeRed AI is running: Confirm the service is listening on the correct port

Services Not Running

  1. Menu option 1 — check which services show as stopped
  2. Menu option 11 — restart individual or all services
  3. If services keep crashing, generate a support bundle (option 14) and contact support
